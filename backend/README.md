## Overview

Python/FastAPI backend for the **RouteIQ вЂ” VRPTW Solver Comparison** project. It runs metaheuristic algorithms (HGS, GLS, ACO, SA, and optionally ILS), serves datasets and parameters, provides solve/compare jobs with streaming logs and route plots, and optional AI/RAG endpoints powered by Google Gemini.

**Two ways to use the backend:**

1. **Web API:** Run `uvicorn app.main:app` and use the React frontend or any HTTP client to trigger solves, stream logs, and fetch results/plots.
2. **Standalone script:** Run `python main.py` to compare all algorithms on one dataset and print a comparison table.

## Problem Definition

The **Vehicle Routing Problem with Time Windows (VRPTW)** involves:

- **Depot:** Central location where all vehicles start and end
- **Customers:** Locations that must be visited with specific demands
- **Time Windows:** Each customer has earliest and latest service start times `[e_i, l_i]`
- **Vehicle Capacity:** Each vehicle has a maximum load capacity
- **Objective:** Minimize total travel distance/cost while respecting all constraints

VRPTW is **NP-Hard** вЂ” no polynomial time algorithm exists for all instances. Metaheuristics are used to find near-optimal solutions in reasonable time.

## Features

- **FastAPI Web App:** REST API for datasets, parameters, solve (single/compare), results, route plot images, optional AI/RAG
- **5 Metaheuristic Algorithms:** HGS, ILS, GLS, ACO, SA вЂ” each in its own module
- **Real-time Streaming:** SSE (Server-Sent Events) for live log streaming during algorithm execution
- **Background Execution:** Solve jobs run in threads; state stored in memory via job_store
- **Route Visualization:** 2D route plots generated by matplotlib
- **AI Parameter Tuning:** Auto-tune via Google Gemini (optional)
- **RAG Q&A:** Ask questions about algorithms using LangChain + ChromaDB
- **Solomon Benchmark:** Standard 56-instance benchmark dataset support
- **Extensible Design:** Easy to add new algorithms or datasets

## Algorithms Implemented

### 1. HGS вЂ” Hybrid Genetic Search

State-of-the-art genetic algorithm combining evolutionary operators with local search. Uses **pyvrp 0.6.3**.

```python
from hgs.solve import solve_with_hgs

routes, cost = solve_with_hgs("dataset/r101.txt", runtime=120)
# routes: list of lists of customer IDs
# cost: total distance (float)
```

**Key idea:** Maintains a population of solutions в†’ crossover + mutation в†’ local search improvement в†’ select best в†’ repeat.

---

### 2. ILS вЂ” Iterated Local Search

Implemented in **pyvrp в‰Ґ0.13**. Used with a separate venv (Option A).

```python
from ils.solve import solve_with_ils

routes, cost = solve_with_ils("dataset/r101.txt", runtime=120)
```

**Key idea:** Perturb current solution в†’ local search в†’ accept/reject в†’ repeat.

---

### 3. GLS вЂ” Guided Local Search

Local search with penalty mechanisms using **Google OR-Tools**.

```python
from gls.solve import solve_with_gls

routes, cost = solve_with_gls("dataset/r101.txt", runtime=120)
```

**Key idea:** Penalize features that cause local optima в†’ modifies landscape в†’ pushes search to new regions.

---

### 4. ACO вЂ” Ant Colony Optimization

Custom implementation of **Multiple Ant Colony System (MACS)**.

```python
from aco.vrptw_base import VrptwGraph
from aco.multiple_ant_colony_system import MultipleAntColonySystem
from aco.solve import get_best_route_from_path

graph = VrptwGraph("dataset/r101.txt", rho=0.1)
macs = MultipleAntColonySystem(
    graph,
    ants_num=30,
    beta=0.9,
    q0=0.9,
    whether_or_not_to_show_figure=False,
    runtime_in_minutes=5,
)
macs.run_multiple_ant_colony_system()
routes = get_best_route_from_path(macs.best_path)
cost = macs.best_path_distance.value
```

**Key idea:** Virtual ants build routes probabilistically guided by pheromone trails. Shorter paths get more pheromone в†’ attract more ants.

---

### 5. SA вЂ” Simulated Annealing

Custom probabilistic optimization implementation.

```python
from sa.instance_loader import load_from_file
from sa.simulated_annealing import sa_algorithm

instance = load_from_file("dataset/r101.txt")
instance.find_initial_solution()
results = sa_algorithm(
    instance,
    temp_start=700,
    update_temp=lambda t: 0.9999 * t,
    stop_criterion=lambda t: t <= 0.01,
    max_runtime_sec=300,
)
routes = results[2][0].get_solution()
cost = results[2][0].get_total_distance()
```

**Key idea:** High temperature в†’ accept worse solutions (exploration). Cool down в†’ converge to better solution. Acceptance: `P = exp(-О”/T)`.

## Parameter Tuning Guide

### ACO Parameters

| Parameter | Range | Recommendation |
|-----------|-------|----------------|
| `ants_num` | 10вЂ“60 | Small: 10-20, Medium: 20-40, Large: 40-60 |
| `beta` | 0.5вЂ“2.0 | 0.9вЂ“1.2 recommended |
| `q0` | 0.0вЂ“1.0 | 0.7вЂ“0.9 recommended |
| `rho` | 0.01вЂ“0.3 | 0.1вЂ“0.15 recommended |

### SA Parameters

| Parameter | Description | Recommendation |
|-----------|-------------|----------------|
| `temp_start` | Initial temperature | Small: 300-500, Large: 800-1200 |
| `update_temp` | Cooling schedule | `0.9999*t` (slow), `0.99*t` (fast) |
| `stop_criterion` | Stop threshold | `t <= 0.01` |

### GLS / HGS / ILS

- Main tunable parameter: **runtime** (seconds)
- Increase runtime for better quality on larger instances

## Code Examples

### Run all algorithms and compare

```python
from hgs.solve import solve_with_hgs
from gls.solve import solve_with_gls
from aco.vrptw_base import VrptwGraph
from aco.multiple_ant_colony_system import MultipleAntColonySystem

# HGS
hgs_routes, hgs_cost = solve_with_hgs(dataset, runtime=runtime)

# GLS
gls_routes, gls_cost = solve_with_gls(dataset, runtime=runtime)

print(f"HGS Cost: {hgs_cost}")
print(f"GLS Cost: {gls_cost}")
```

### Call the web API from Python

```python
import requests, time

BASE = "http://localhost:8000/api"

# Start solve
r = requests.post(f"{BASE}/solve/hgs", json={"dataset": "r101", "runtime": 120})
job_id = r.json()["job_id"]

# Poll result
r = requests.get(f"{BASE}/results/{job_id}")
while r.json().get("status") == "running":
    time.sleep(2)
    r = requests.get(f"{BASE}/results/{job_id}")

result = r.json().get("result")
print(f"Cost: {result['cost']}, Routes: {len(result['routes'])}")
```
